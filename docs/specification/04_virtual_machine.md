# Спецификация виртуальной машины

Виртуальная машина (VM) является стековой: все вычисления выполняются через стек значений (`EvaluationStack`), без регистров общего назначения.  
Компилятор (`PsVmCodegen`) генерирует линейный байткод (`List<Instruction>`), который исполняется классом `PsVm`.

## Компоненты VM

- `PsVm` — исполнительный цикл `fetch/decode/execute`.
- `Program` (`IReadOnlyList<Instruction>`) — последовательность байткода.
- `InstructionPointer` (`_instructionPointer`) — индекс текущей инструкции.
- `EvaluationStack` (`Stack<Value>`) — стек операндов и промежуточных результатов.
- `VariablesTable` (`_variables`) — **текущая** таблица локальных переменных; у каждой таблицы есть ссылка `Parent` на внешнюю таблицу. Цепочка `Parent` образует лексические области видимости **внутри одного вызова функции** (см. `PushVars` / `PopVars`).
- `FrameStack` (`_frameStack`) — стек ссылок на таблицы переменных (`Stack<VariablesTable?>`). Хранит «корень» цепочки `_variables` вызывающего контекста на время выполнения вызываемой функции; используется инструкциями `Call` / `Return` (см. п. 5).
- `BuiltinFunctions` — реализация встроенных функций.
- `Environment` — внешний ввод-вывод.
- `CallStack` (`_callStack`) — стек целых (`Stack<int>`), в котором хранятся **возвратные адреса** для инструкции `Call`: после стандартного увеличения `InstructionPointer` в цикле исполнения в стек кладётся индекс следующей инструкции после `Call`; при `Return` адрес снимается и подставляется в `InstructionPointer` (см. п. 5).
- `ExitCode` (`_exitCode`) — код завершения, возвращаемый `RunProgram()`.

## Цикл выполнения

1. Взять `instruction = Program[InstructionPointer]`.
2. Увеличить `InstructionPointer`.
3. Выполнить действие согласно `instruction.Code`.
4. Продолжать до `Halt`.

Проверки перед запуском:

- программа не пуста;
- в списке инструкций есть **хотя бы одна** `Halt`; проверяется только **наличие** такой инструкции в `Program`, а не то, что она последняя. После изменения кодогенерации при поддержке пользовательских функций (`Call`/`Return`) и общем линейном списке инструкций `Halt` может располагаться не в конце программы.

## Формат инструкций и значения

Инструкция содержит:

- `Code: InstructionCode`;
- `Operand: Value` (присутствует всегда; если операнд не нужен, используется `Value.Unit`).

Поддерживаемые типы `Value`:

- `int` (`long`);
- `float` (`double`);
- `str` (`string`);
- `bool` (`bool`);
- `unit`.

## Набор реализованных инструкций

Обозначения:

- `EVAL[^1]` — вершина стека;
- `EVAL[^2]` — элемент под вершиной.

## Инструкции VM

### 1) Стек и переменные

1. `Push [Value]` — кладет значение в стек.
2. `Pop` — снимает и отбрасывает вершину стека.
3. `DefineLocal [Name]` — снимает значение со стека и объявляет новую переменную в текущей таблице.
4. `LoadLocal [Name]` — загружает значение переменной в стек.
5. `StoreLocal [Name]` — снимает значение и присваивает его уже существующей переменной (поиск идёт по цепочке `Parent`; присваивание — в ближайшей таблице, где имя уже объявлено).
6. `PushVars` — создаёт новую таблицу `VariablesTable` с `Parent = _variables` и делает её текущей (`_variables = new VariablesTable(_variables)`).
7. `PopVars` — снимает текущую таблицу: `_variables = _variables.Parent`.

**Лексические области в байткоде.** Кодогенератор (`PsVmCodegen`) оборачивает каждый блок кода парой `PushVars` / `PopVars` (методы `PushScope` / `PopScope`):

- тело функции (включая объявление параметров);
- составной оператор `{ ... }` (`BlockStatement`);
- тело цикла `for` (отдельно от области итератора/счётчика).

Вложенные блоки дают вложенную цепочку `Parent`. При выходе из блока `PopVars` уничтожает локальные объявления этого блока; переменные внешних областей остаются доступны через `Parent`.

### 2) Арифметика

8. `Add` — `left + right`.
9. `Subtract` — `left - right`.
10. `Multiply` — `left * right`.
11. `Divide` — `left / right` (`int/int` дает целочисленный результат, иначе `float`).
12. `Modulo` — `left % right` (для `int` и `float`).
13. `Power` — `pow(left, right)` (`Math.Pow`).
14. `Negate` — унарный минус.

Правило бинарной числовой операции: если хотя бы один операнд `float`, результат `float`; иначе `int`.

### 3) Логика и сравнения (итерация №4)

15. `Not` — логическое отрицание `!operand`.
16. `Less` — сравнение `<`.
17. `LessOrEqual` — сравнение `<=`.
18. `Equal` — сравнение `==`.
19. `NotEqual` — сравнение `!=`.

Операторы `>` и `>=` в байткоде генерируются через перестановку операндов и инструкции `Less` / `LessOrEqual`.

Операторы `&&` и `||` **не имеют** отдельных инструкций VM. Кодогенерация реализует их через **короткое замыкание** и переходы:

- `&&` (`GenerateLogicalAndCode`): вычисляется левый операнд; `JumpIfFalse` в блок с результатом `false`; иначе вычисляется правый операнд, результат приводится к `bool` через сравнение с `false` (`Push false`, `NotEqual`); объединение веток — `Jump` в общий `final` блок.
- `||` (`GenerateLogicalOrCode`): вычисляется левый операнд; `JumpIfTrue` в блок с результатом `true`; иначе — аналогичная проверка правого операнда; объединение в `final` блок.

### 4) Переходы и ветвления if/else (итерация №4)

20. `Jump [Address]` — безусловный переход.
21. `JumpIfTrue [Address]` — снимает `bool`-условие; переходит при `true`.
22. `JumpIfFalse [Address]` — снимает `bool`-условие; переходит при `false`.

Кодогенерация `if/else`:

- условие вычисляется и кладется в стек;
- генерируется условный переход в `else`/`final` блок;
- затем `then`-ветка;
- затем переход в `final`;
- затем `else`-ветка (если есть);
- продолжение в `final` блоке.

### 5) Вызовы пользовательских функций (итерация №5)

Добавлены инструкции **`Call`** и **`Return`**. Для них используются **`_callStack`** (адреса возврата) и **`_frameStack`** (сохранение цепочки переменных вызывающего), см. раздел «Компоненты VM».

23. `Call [Address]` — вызов функции по абсолютному индексу инструкции `Address` в `Program`. После стандартного шага цикла (чтение инструкции и увеличение `InstructionPointer`) VM:

- кладёт текущий `InstructionPointer` в `_callStack` (адрес возврата);
- кладёт текущую `_variables` в `_frameStack`;
- устанавливает `_variables = null`;
- выполняет переход: `InstructionPointer = Address`.

Первая таблица локальных переменных **вызываемой** функции создаётся не в `Call`, а в начале её байткода инструкцией `PushVars` (кодогенерация вызывает `PushScope` для тела функции). Параметры объявляются через `DefineLocal` уже внутри этой области.

24. `Return` — возврат из подпрограммы:

- снимает `EVAL[^1]` как возвращаемое значение;
- восстанавливает `_variables` из вершины `_frameStack` (`Pop`);
- восстанавливает `InstructionPointer` из вершины `_callStack` (`Pop`);
- кладёт возвращаемое значение обратно в `EvaluationStack`.

Перед `Return` кодогенерация для функции должна завершить её лексическую область (`PopVars` / `PopScope`), чтобы в момент возврата внутренняя цепочка `PushVars` не «перетекала» в вызывающий контекст — восстанавливается только сохранённый при `Call` корень `_variables` вызывающего.

### 6) Встроенные функции

25. `CallBuiltin [Code]` — вызывает встроенную функцию по `BuiltinFunctionCode`.

Поддерживаемые коды встроенных функций:

- `Print`, `PrintI`, `PrintF`, `PrintB`;
- `Input`;
- `ItoS`, `FtoS`, `ItoF`, `FtoI`, `StoI`, `StoF`;
- `SConcat`, `SubStr`, `StrLen`.

### 7) Завершение

26. `Halt` — останавливает VM, снимает `EVAL[^1]` как `ExitCode` (`int`) и возвращает его из `RunProgram()`.

## Управление областями видимости (сводка)

Два механизма дополняют друг друга:

| Механизм | Структура | Когда |
|----------|-----------|--------|
| `PushVars` / `PopVars` | цепочка `VariablesTable.Parent` | вход/выход из блока кода внутри одной функции |
| `Call` / `Return` + `_frameStack` | сохранение и восстановление корня `_variables` | вход/выход из вызова пользовательской функции |

Схема при вложенном блоке и вызове:

```text
Вызывающий:  _variables → [блок A] → [блок B]
Call:        _frameStack.push(корень вызывающего); _variables = null
Функция:     PushVars → [параметры и тело] → … → PopVars → Return
Return:      _variables = _frameStack.pop(); продолжение с адреса из _callStack
```
